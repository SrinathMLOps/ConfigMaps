# Visual Assets for ConfigMaps Project

This directory should contain the following images:

## 1. configmaps-banner.png
A banner image showing:
- Kubernetes logo
- ConfigMap icon
- Title: "Kubernetes ConfigMaps - Complete Guide"
- Subtitle: "Separate Configuration from Code"

Suggested dimensions: 1200x400px

## 2. architecture.png
Architecture diagram showing:
- Developer → kubectl
- API Server
- etcd storage
- Kubelet on worker nodes
- Pods with containers
- ConfigMap injection (env, volume, envFrom)

## 3. mindmap.png
Visual mind map showing:
- Central node: "ConfigMap"
- Branches:
  - What it is
  - Why it exists
  - 3 consumption types
  - Real-world scenarios
  - Best practices

## How to Create These Images

You can create these using:
- Draw.io (https://app.diagrams.net/)
- Excalidraw (https://excalidraw.com/)
- Canva (https://www.canva.com/)
- Figma (https://www.figma.com/)

## Placeholder Images

Until you create custom images, you can use:
- Kubernetes official icons
- Simple text-based diagrams
- ASCII art in markdown

## Color Scheme

Recommended colors matching Kubernetes branding:
- Primary: #326CE5 (Kubernetes blue)
- Secondary: #FFFFFF (White)
- Accent: #00D1B2 (Teal)
- Text: #333333 (Dark gray)
